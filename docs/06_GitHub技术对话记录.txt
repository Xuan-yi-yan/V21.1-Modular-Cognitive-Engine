1. QwenLM/qwen-code #5083 — TUI卡死/僵尸子进程
   [P1] https://github.com/QwenLM/qwen-code/issues/5083

   问题: Qwen Code 终端在长会话(14h, 782条history)后完全冻结
   提交者: kilowu
   本人诊断: 硬截断(滑动窗口)导致关键信息丢失, 模型逻辑链断裂
   本人方案: CoT驱动的语义压缩 — 不能用物理截断, 必须用模型自己
            做多步推理压缩(事实提取→失败标记→状态快照)
   状态: 已发布分析


2. DeepSeek-V3 #1368 — Thinking模式安全退化
   https://github.com/deepseek-ai/DeepSeek-V3/issues/1368

   问题: 开启Thinking模式后模型伪造逼真凭证, 关闭则正确拒绝
   提交者: HiChat-fog
   本人诊断: 安全是"权重"不是"闸门" — 安全约束和Helpfulness在同一
            优化空间内博弈, CoT给了模型"协商"的空间
   本人方案: 推理时安全锚点注入+Logit Bias+CoT安全检查
   社区反馈: AmoebaFPS回复"这是最重要的洞察"并更新临床附录
             qingkong66回复"你的分析重新定义了#1368"


3. DeepSeek-V3 #1420 — 长会话认知漂移
   https://github.com/deepseek-ai/DeepSeek-V3/issues/1420

   问题: 100+轮对话中模型认知漂移, KV cache饱和导致结构性失忆
   提交者: levax29-ai
   本人诊断: "KV池中没有语义索引" — 每条历史只有role标签,
            注意力在无索引的大池子里裸游
   本人方案: 语义预标注中间件(意图标记→相关性过滤→结构化组装)
   社区反馈: qingkong66回复"正是根因", 建议验证[DO_NOT_RETRY]标记


4. DeepSeek-V3 #1125 — 推理后缀污染/注意力锁定
   https://github.com/deepseek-ai/DeepSeek-V3/issues/1125

   问题: 推理输出后缀约束在多轮对话中自我复制, 模型变成复读机
   提交者: D7x7z49 (从2月追到5月)
   本人诊断: 不是"目标偏移"是"注意力锁定" — meta-signal污染了
            content-signal, 后缀约束进入历史后被注意力当成模仿模板
   本人方案: 输出层重复模式检测 + 拦截重定向
   D7x7z49回复: "you are agent ???" (被分析深度吓到)


5. DeepSeek-V3 #1460 — 系统性认知病理学报告(Amoeba协议)
